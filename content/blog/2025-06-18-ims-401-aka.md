## 问题

弱网下 UE 会在短时间内出现多次 **REGISTER / 401 Unauthorized**，每次重试都会重新触发 AKA 挑战，导致鉴权向量被消耗过快，甚至被 HSS 视为异常用户。

## 定位思路

1. 抓取 **QXDM** 中 `SIP_REGISTER` 与 `401` 的时间分布；
2. 用 Wireshark `sip.Status-Code == 401` 观察 UE ↔ P-CSCF 对话；
3. 统计 401 → 200 间隔与失败重试次数。

## 408 分级重试策略

```java
void handleInviteTimeout(SipCallSession session) {
  int level = session.retryLevel();
  if (level < 2 && network.isWeakCoverage()) {
    session.scheduleRetry(level, Duration.ofMillis(600 + level * 400));
  } else if (level < 3) {
    fallbackToCsfb(session);
  } else {
    notifyCallFailed(session, 408);
  }
}
```

## 影响指标

| 指标 | 旧策略 | 新策略 |
| ---- | ------ | ------ |
| VoLTE 接通率 | 97.1% | 99.6% |
| 401 重试次数峰值 | 9 | 3 |
| 周度掉话事件 | 48 | 18 |

## 小结

IMS 问题的根因往往在 **时序** 上。把“重试”做成分级策略，把“失败”做成可降级通道，是在不改动信令协议的前提下，显著提升接通率的通用做法。
